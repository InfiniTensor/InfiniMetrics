QUICK START:
═════════════

1. Build:
   $ cd benchmarks/hardware/cuda-memory-benchmark
   $ ./build.sh

2. Run all tests:
   $ ./build/cuda_perf_suite --all

3. Run specific tests:
   $ ./build/cuda_perf_suite --stream     # STREAM benchmark only
   $ ./build/cuda_perf_suite --memory     # Memory tests only
   $ ./build/cuda_perf_suite --cache      # Cache tests only
   $ ./build/cuda_perf_suite --comm       # Multi-GPU tests

4. Custom parameters:
   $ ./build/cuda_perf_suite --stream --array-size 134217728 --iterations 20

DESIGN HIGHLIGHTS:
══════════════════

1. No Code Duplication
   - 100% new implementation
   - Different architecture
   - Modern design patterns

2. Object-Oriented Design
   - Base class: PerformanceTest
   - Derived classes for each test
   - Polymorphic execution

3. Template-Based Kernels
   - Type-safe implementations
   - Compile-time optimization
   - Code reuse

4. RAII Wrappers
   - CudaDeviceBuffer<T>
   - CudaPinnedBuffer<T>
   - CudaStream
   - Automatic cleanup

5. Statistical Framework
   - PerformanceMetrics class
   - Trimmed mean (removes outliers)
   - Coefficient of variation
   - Median/min/max calculations

EXTENSIBILITY:
═══════════════

Adding new tests is straightforward:

1. Inherit from PerformanceTest base class
2. Implement virtual methods:
   - name()
   - description()
   - initialize()
   - cleanup()
   - run_warmup()
   - run_single_test()
3. Add to main.cu
4. Rebuild

No modifications to framework code needed!
